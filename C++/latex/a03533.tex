\doxysubsubsection*{The {\ttfamily \textbackslash{}code} tag should be used for code block examples under 10 lines.}

\href{https://www.doxygen.nl/manual/commands.html\#cmdcode}{\texttt{ https\+://www.\+doxygen.\+nl/manual/commands.\+html\#cmdcode}}

Here is an example for using the {\ttfamily \textbackslash{}code} tag\+:

/$\ast$! 
\begin{DoxyCode}{0}
\DoxyCodeLine{// Use the '//' comment style.}
\DoxyCodeLine{std::cout << "{}Here is a test statement."{} << std::endl;}
\DoxyCodeLine{}
\DoxyCodeLine{// You may indent your code sample as needed.}
\DoxyCodeLine{struct my\_point \{}
\DoxyCodeLine{    double m\_x;}
\DoxyCodeLine{    double m\_y;}
\DoxyCodeLine{\};}

\end{DoxyCode}
 $\ast$/

Follow these steps to ensure consistency within the codebase\+: 0. Start an example block with {\ttfamily /$\ast$!}, then hit {\bfseries{ENTER}} to go to the next line.
\begin{DoxyEnumerate}
\item Hit {\bfseries{TAB}} once, then use the {\ttfamily \textbackslash{}code} tag. Then, hit {\bfseries{ENTER}}.
\item Hit {\bfseries{TAB}} twice, then begin your code example. You may indent as necessary for your example.
\item When your example is finished, hit {\bfseries{ENTER}}, then {\bfseries{BACKSPACE}} so that you are inline with the {\ttfamily \textbackslash{}code} tag.
\item Use the {\ttfamily \textbackslash{}endcode} tag to end your example.~\newline
 Then hit {\bfseries{ENTER}}, then hit {\bfseries{BACKSPACE}} as many times as necessary so that you are inline with {\ttfamily /$\ast$!}.
\item Finally, end the example block with {\ttfamily $\ast$/}.
\end{DoxyEnumerate}
\begin{DoxyItemize}
\item Note that any comments that occur within the sample must begin with {\ttfamily //}, followed by a whitespace.
\end{DoxyItemize}

This procedure strikes the best compromise between\+:
\begin{DoxyItemize}
\item Making sure that comments work within a code example
\item Not having to remove {\ttfamily //} or {\ttfamily ///} per line of the example
\end{DoxyItemize}

~\newline
This ensures ease of viewing~\newline
 for anyone reading the source code -- ~\newline
 and also assists Doxygen with the~\newline
 formatting process for the HTML export.

You may now use {\bfseries{\textquotesingle{}Run Doxygen\textquotesingle{}}} to export your documentation.~\newline
 (There are no extra steps, unlike with the snippet section)

\doxysubsection*{Adding images for headers/sources, and markdown files ({\ttfamily .md})}

For {\bfseries{headers/sources}},~\newline
 we will be using the {\ttfamily \textbackslash{}image} tag to add images.~\newline
 \href{https://www.doxygen.nl/manual/commands.html\#cmdimage}{\texttt{ https\+://www.\+doxygen.\+nl/manual/commands.\+html\#cmdimage}}

For {\bfseries{markdown files ({\ttfamily .md})}},~\newline
 we will be using the markdown syntax for images.~\newline
 \href{https://www.doxygen.nl/manual/markdown.html\#md_images}{\texttt{ https\+://www.\+doxygen.\+nl/manual/markdown.\+html\#md\+\_\+images}}

In {\bfseries{dhart}},~\newline
 we have a subdirectory named {\bfseries{docs}}.~\newline
 ({\ttfamily dhart\textbackslash{}docs})~\newline


The Doxyfile will already be configured to have it\textquotesingle{}s {\ttfamily IMAGE\+\_\+\+PATH}~\newline
 configured such that its working directory for images begins at~\newline
 {\ttfamily dhart\textbackslash{}docs}.~\newline
 When we proceed to later steps, and we are to insert the path of an image file,~\newline
 assume that the working directory for images begins at~\newline
 {\ttfamily dhart\textbackslash{}docs.}

For example, if a file is saved at\+:~\newline
 {\ttfamily dhart\textbackslash{}docs\textbackslash{}C++\textbackslash{}images\textbackslash{}spatialstructures\textbackslash{}node\textbackslash{}test-\/image\+\_\+node-\/h.\+png},~\newline
 you will specify the path\+:~\newline
 {\ttfamily C++\textbackslash{}images\textbackslash{}spatialstructures\textbackslash{}node\textbackslash{}test-\/image\+\_\+node-\/h.\+png}.

Notice that the {\bfseries{dhart/docs}} portion is omitted when specifying the image,~\newline
 since our Doxyfile will already its {\ttfamily IMAGE\+\_\+\+PATH} configured to begin at {\bfseries{dhart/docs}}.

{\bfseries{Step 0\+:}}~\newline
 Begin by saving copy of {\bfseries{image\+\_\+file}},~\newline
 which is the filename of the desired image,~\newline
 into the following directory\+:~\newline


{\ttfamily dhart\textbackslash{}docs\textbackslash{}language\textbackslash{}img\textbackslash{}enclosing\+\_\+folder\textbackslash{}module\+\_\+name}

{\ttfamily \texorpdfstring{$<$}{<}b\texorpdfstring{$>$}{>}language\texorpdfstring{$<$}{<}/b\texorpdfstring{$>$}{>}},~\newline

 and {\ttfamily \texorpdfstring{$<$}{<}b\texorpdfstring{$>$}{>}module\+\_\+name\texorpdfstring{$<$}{<}/b\texorpdfstring{$>$}{>}} are {\bfseries{placeholders}}.~\newline


{\ttfamily images} will be a subdirectory within {\ttfamily \texorpdfstring{$<$}{<}b\texorpdfstring{$>$}{>}language\texorpdfstring{$<$}{<}/b\texorpdfstring{$>$}{>}}.
