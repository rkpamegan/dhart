\doxysubsection{Detailed Description}
A graph usable with the Boost\+Graph\+Library. 

Contains a CSR in boost created from a graph in DHARTAPI. This is necessary for using any of the Boost\+Graph\+Library functions.

\begin{DoxyInvariant}{Invariant}
This class will always carry a valid boost graph.
\end{DoxyInvariant}
\begin{DoxyRemark}{Remarks}
None of the memebers of this class should need to be modified after the class is created. This is intended to be a pseudo-\/wrapper to make the \mbox{\hyperlink{a01551}{HF\+::\+Spatial\+Structures\+::\+Graph}} usable with boost. You can read more about the boost graph libary in the official boost documentation\+: \href{https://www.boost.org/doc/libs/1_73_0/libs/graph/doc/quick_tour.html}{\texttt{ https\+://www.\+boost.\+org/doc/libs/1\+\_\+73\+\_\+0/libs/graph/doc/quick\+\_\+tour.\+html}} Boost\textquotesingle{}s graphs can be difficult to use due to the extensive use of templates. This limits their usage outside of the HF\+::\+Pathfinder, so there aren\textquotesingle{}t any functions to interact with the boost graph after creation.
\end{DoxyRemark}
\begin{DoxySeeAlso}{See also}
\mbox{\hyperlink{a01551}{HF\+::\+Spatial\+Structures\+::\+Graph}} for a graph that\textquotesingle{}s better supported in DHARTAPI.

Every edge and node in the graph is used to create a graph in Boost. This will also allocate space equal to the number of nodes in g for this class\textquotesingle{}s p and d arrays for use in Find\+Path as an optimization.

1) If {\ttfamily cost\+\_\+type} is specified, and is not an empty string, then {\ttfamily cost\+\_\+type} must be the key of an already created cost in {\ttfamily graph}.
